> **版本**: v3.0 | **状态**: 正式发布 | **日期**: 2026-04-24  
> **适用范围**: AITestPro 全功能模块  
> **目标读者**: 产品经理 / 业务分析师 / 测试团队 / 开发团队

## 第 1 章 产品概述

### 1.1 产品定位

AITestPro 是一款面向测试团队的 AI 驱动自动化测试管理平台，核心价值在于利用大语言模型技术大幅降低测试用例编写成本，同时提供完整的测试执行、结果分析和质量洞察能力，帮助团队以更高效率保障软件质量。

### 1.2 目标用户

| 角色 | 主要诉求 | 核心使用场景 |
|------|---------|-------------|
| 管理员 | 管理账号、权限与系统配置 | 账号管理、角色配置、集成设置 |
| 测试工程师 | 高效生成与执行测试用例 | AI 用例生成、执行调度、结果分析 |
| 测试负责人 | 掌握质量状况、把控风险 | 仪表盘监控、智能分析、测试报告 |
| 开发人员 | 查看失败用例与缺陷详情 | 测试结果、缺陷分析 |

### 1.3 功能模块总览

| 模块分组 | 功能模块 | 核心职责 |
|---------|---------|---------|
| 概览 | 仪表盘 | 展示全局测试指标、快速入口与环境状态 |
| 概览 | 项目列表 | 管理测试项目的创建、查看与配置 |
| AI 智能 | AI 用例生成 | 基于需求文档/自然语言，AI 自动生成测试用例 |
| AI 智能 | 智能分析 | 基于历史数据，AI 提供质量洞察与风险预警 |
| 测试执行 | 执行任务 | 管理测试任务的启动、监控与停止 |
| 测试执行 | 定时任务 | 配置定时自动触发的测试计划 |
| 测试执行 | 执行环境 | 管理测试环境资源与配置 |
| 报告中心 | 测试结果 | 实时监控、历史记录、趋势分析与缺陷清单 |
| 报告中心 | 测试报告 | 查看、筛选与导出标准化测试报告 |
| 系统管理 | 账号管理 | 管理用户账号、角色与权限配置 |
| 系统管理 | 集成设置 | 配置 CI/CD、通知及第三方工具对接 |

### 1.4 全局交互规范

- **全局搜索**: 导航栏中央提供统一搜索入口（快捷键 ⌘K），支持跨项目、用例、报告的关键词搜索
- **通知中心**: 导航栏通知图标汇聚系统消息，包含测试完成、执行失败、定时任务触发等实时推送，支持一键全部已读
- **用户菜单**: 右上角头像展开下拉菜单，提供快速跳转"我的项目"、"执行记录"、"账号设置"及退出登录
- **主题切换**: 导航栏右侧提供明色/深色主题切换，用户偏好实时生效
- **AI 助手**: 导航栏左侧提供 AI 助手快捷入口，支持侧拉抽屉式交互
- **侧边栏折叠**: 侧边栏支持一键收起/展开，收起后以图标形式保留导航能力

## 第 2 章 登录与认证模块

### F-AUTH-001 用户登录 [P0]

| 属性 | 说明 |
|------|------|
| 功能概述 | 用户通过用户名和密码完成身份验证，成功后进入平台主界面 |
| 触发方式 | 用户访问系统首页，自动跳转登录页；或登录态失效时重定向 |
| 操作流程 | 1. 输入用户名 2. 输入密码 3. 可选勾选"记住我" 4. 点击"登录"按钮 |
| 成功结果 | 通过身份验证后，页面以动画过渡方式跳转至仪表盘页面 |
| 失败处理 | 用户名或密码错误时显示明确错误提示，不清空已输入内容 |

**业务规则:**
1. 用户名和密码均为必填项，任一为空时"登录"按钮不可提交
2. 勾选"记住我"后，下次访问自动填充用户名
3. 系统提供演示账号（admin / admin123）供体验使用

### F-AUTH-002 密码可见性切换 [P1]

| 属性 | 说明 |
|------|------|
| 功能概述 | 用户可在密码输入框旁点击眼睛图标，在密文和明文之间切换 |
| 触发方式 | 点击密码输入框右侧的眼睛图标 |
| 业务规则 | 切换操作不影响输入框中已输入的内容，不触发表单提交 |

### F-AUTH-003 忘记密码 [P1]

- 点击登录页"忘记密码？"链接，弹出提示："请联系管理员重置密码"
- 密码自助重置功能暂未开放，统一通过管理员在账号管理模块执行重置

### F-AUTH-004 自动登录检测 [P0]

- 系统在页面加载时自动检测用户是否处于已登录状态
- 检测到有效登录态时直接跳过登录页进入仪表盘
- 登录态存储在浏览器 localStorage，关闭浏览器且未勾选"记住我"时登录态不保留

### F-AUTH-005 退出登录 [P0]

- 点击导航栏右上角用户头像 → 展开 → 点击"退出登录"
- 登录态清除，页面跳转回登录页
- "记住我"缓存的用户名仍保留，密码不保留

## 第 3 章 仪表盘模块

### F-DASH-001 核心指标卡片展示 [P0]

页面顶部以卡片形式展示 4 项核心测试指标：
1. 测试用例总数（附环比趋势箭头）
2. 用例通过率（附环比趋势箭头）
3. 今日执行次数（附环比趋势箭头）
4. 待处理缺陷数（附环比趋势箭头）

### F-DASH-002 场景导航模块入口 [P1]

以可视化模块卡片形式呈现平台 4 大核心场景，并作为快速跳转入口：
1. AI 测试用例生成 → 跳转"AI 用例生成"页面
2. 自动化测试执行 → 跳转"执行任务"页面
3. 测试结果展示 → 跳转"测试结果"页面
4. 智能测试报告 → 跳转"测试报告"页面

### F-DASH-003 测试执行趋势图 [P1]

- 以折线图展示近 7 天/30 天的测试执行趋势
- 支持按天/周/月粒度切换
- 包含执行次数、通过率、失败数三条数据线

### F-DASH-004 最近执行列表 [P1]

- 展示最近 5 条测试执行记录
- 每条显示：项目名、执行状态（通过/失败/进行中）、执行时间、持续时间
- 支持点击查看执行详情

### F-DASH-005 执行环境状态展示 [P2]

- 展示各测试环境的在线/离线状态
- 显示环境名称、最近活动时间、资源使用率

## 第 4 章 项目管理模块

### F-PROJ-001 项目列表展示 [P0]

- 以卡片/列表视图展示所有测试项目

### F-PROJ-002 新建项目 [P0]

- 点击"新建项目"按钮弹出模态框
- 填写：项目名称、项目描述、所属系统、负责人
- 创建成功后自动出现在列表中

### F-PROJ-003 项目搜索与筛选 [P1]

- 支持按项目名称关键词搜索
- 支持按状态（进行中/已完成/暂停）筛选

### F-PROJ-004 项目操作（编辑/删除）[P1]

- 每个项目卡片提供"编辑"和"删除"操作
- 编辑弹出模态框，删除需二次确认

## 第 5 章 AI 用例生成模块

### F-GEN-001 输入方式选择 [P0]

- 提供多种输入方式切换：文档上传、自然语言、Swagger/OpenAPI、关联项目
- 选择后切换对应的输入区域

### F-GEN-002 文档上传解析 [P0]

- 支持拖拽或点击上传文件
- 支持 .docx / .pdf / .xlsx / .md / .html 格式
- 上传后显示文件信息卡片（文件名、大小、上传时间）
- 自动调用 AI 解析文档，提取需求点
- 显示解析进度条和解析结果（需求点数量、摘要）
- 支持移除已上传文件重新上传

### F-GEN-003 自然语言需求输入 [P0]

- 提供文本输入区域
- 用户可直接输入自然语言描述的需求
- 支持提交后调用 AI 生成测试用例

### F-GEN-004 用例层级配置 [P0]

- 提供 5 级执行粒度选择：系统级 > 模块级 > 功能级 > 用例级 > 接口级
- 每个层级可选择/自定义具体项目

### F-GEN-005 生成配置 [P1]

- 用例数量：自动 / 指定数量
- 场景类型：全部 / 正向 / 反向 / 边界值 / 异常
- 优先级：全部 / 高 / 高+中

### F-GEN-006 高级配置 [P2]

- 入库模式：全自动 / 人工审核 / 混合
- 测试类型标签选择

### F-GEN-007 开始生成 [P0]

- 点击"开始生成"按钮触发 AI 生成
- 校验是否已提供输入源（文档或自然语言）
- 显示生成进度条：分析需求 → 提取要点 → 逐条生成
- 生成完成后提示

### F-GEN-008 生成预览与确认 [P0]

- 右侧预览区逐条展示生成的测试用例
- 每条用例包含：编号、标题、描述、优先级、场景标签、所属模块
- 支持操作：通过 / 拒绝 / 编辑
- 通过的用例标记绿色，拒绝的标记红色

### F-GEN-009 生成历史记录 [P2]

- 记录每次生成的历史，支持查看和重新加载

## 第 6 章 执行任务模块

### F-EXEC-001 执行层级选择 [P0]

- 提供 5 级执行粒度选择（系统/模块/功能/用例/接口）
- 层级选择器支持自定义项目添加

### F-EXEC-002 执行环境选择 [P0]

- 显示可用测试环境列表
- 每个环境显示状态（在线/离线）和资源信息
- 支持选择目标执行环境

### F-EXEC-003 立即执行 [P0]

- 点击"立即执行"按钮启动测试
- 支持串行/并行执行模式切换
- 可配置并发数（1-1000）、超时时间、重试次数
- 执行过程中显示实时日志

### F-EXEC-004 执行历史列表 [P0]

- 展示所有执行记录
- 每条记录显示：执行时间、状态、用例数、通过率、持续时间
- 支持筛选和查看详情

### F-EXEC-005 执行任务停止 [P1]

- 执行中任务可点击"停止"按钮终止
- 停止后保留已执行部分的结果

## 第 7 章 定时任务模块

### F-SCH-001 定时任务列表 [P1]

## 第 12 章 账号管理模块

### F-ACC-001 账号列表展示 [P0]

- 展示所有用户账号
- 显示：用户名、姓名、角色、状态（启用/禁用）、最后登录时间

### F-ACC-002 新增账号 [P0]

- 填写：用户名、初始密码、姓名、角色、邮箱、手机号、部门
- 创建后可通过初始密码首次登录

### F-ACC-003 编辑账号 [P1]

- 修改用户信息，不可修改用户名

### F-ACC-005 启用/禁用账号 [P1]

- 禁用后用户无法登录，但数据保留

### F-ACC-006 删除账号 [P1]

- 需二次确认，删除后数据不可恢复

### F-ACC-007 角色权限配置 [P0]

- 预设角色：管理员、测试工程师、测试负责人、开发人员（只读）

## 附录 B 术语表

| 术语 | 说明 |
|------|------|
| 测试用例 (Test Case) | 描述单个测试场景的文档，包含步骤、预期结果和优先级 |
| 用例层级 | 测试颗粒度划分：系统级 > 模块级 > 功能级 > 用例级 > 接口级 |
| 执行任务 | 一次测试执行的完整过程，包括配置、运行、结果收集 |
| 入库模式 | 生成的用例进入正式用例库的审核方式：全自动/人工审核/混合 |
| 执行环境 | 测试脚本实际运行所在的服务器/容器资源集合 |
| 缺陷 (Defect) | 测试执行中发现的软件问题，按严重级别分为严重/一般/轻微 |
| 覆盖率 | 已测试内容占全部预期测试内容的比例，分需求/功能/接口覆盖率 |
| AI 洞察 | AI 基于历史测试数据自动生成的质量分析结论和建议 |
| P0/P1/P2 | 功能优先级：P0 为必须实现，P1 为应当实现，P2 为可选实现 |
